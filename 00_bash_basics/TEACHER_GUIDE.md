## 📖 อภิธานศัพท์คำสั่ง (Command Glossary) — "ตัวย่อมาจากอะไร"

> 🎤 **เคล็ดลับการสอน:** ชื่อคำสั่งบน Linux ส่วนใหญ่เป็น "ตัวย่อ" ของคำอังกฤษ
> พอบอกที่มา นักเรียนจะจำได้เร็วขึ้นมาก เช่น `grep` ฟังดูประหลาด แต่พอรู้ว่าย่อมาจาก
> *"Global Regular Expression Print"* ก็เห็นภาพทันทีว่ามันคือ "ค้นหาแล้วพิมพ์ออกมา"

**คำสั่งพื้นฐาน & ตัวตน (บท 01)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `whoami` | "who am i" | ฉัน login เป็น user อะไร |
| `hostname` | host name | ชื่อเครื่อง/เซิร์ฟเวอร์ |
| `date` | date | วันและเวลาปัจจุบัน |
| `echo` | echo (สะท้อนเสียง) | พิมพ์ข้อความ/ค่าตัวแปรออกจอ |

**Navigation (บท 02)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `pwd` | **P**rint **W**orking **D**irectory | บอก path ที่เรายืนอยู่ตอนนี้ |
| `ls` | **L**i**s**t | แสดงรายชื่อไฟล์/โฟลเดอร์ |
| `cd` | **C**hange **D**irectory | เดินเข้า/ออกโฟลเดอร์ |
| `tree` | tree (ต้นไม้) | แสดงโครงสร้างโฟลเดอร์เป็นแผนผัง |

**`ls` ใช้กับ flag อะไรได้บ้าง (ที่ใช้บ่อย ~12 แบบ)**

| Flag | ย่อมาจาก | ผลลัพธ์ |
|------|----------|---------|
| `-l` | **L**ong | แสดงแบบยาว: สิทธิ์ เจ้าของ ขนาด วันที่ |
| `-a` | **A**ll | แสดงไฟล์ซ่อน (ขึ้นต้นด้วย `.`) ทั้งหมด รวม `.` และ `..` |
| `-A` | **A**lmost all | แสดงไฟล์ซ่อน แต่ไม่เอา `.` และ `..` |
| `-h` | **H**uman-readable | ขนาดเป็น K/M/G (ใช้คู่ `-l`) |
| `-t` | **T**ime | เรียงตามเวลาแก้ไขล่าสุด (ใหม่สุดก่อน) |
| `-r` | **R**everse | กลับลำดับการเรียง |
| `-S` | **S**ize | เรียงตามขนาดไฟล์ (ใหญ่ก่อน) |
| `-R` | **R**ecursive | ไล่แสดงทุกโฟลเดอร์ย่อย |
| `-1` | one | หนึ่งไฟล์ต่อหนึ่งบรรทัด |
| `-F` | classi**F**y | เติมสัญลักษณ์: `/`=โฟลเดอร์, `*`=ไฟล์รันได้, `@`=symlink |
| `-d` | **D**irectory | แสดงตัวโฟลเดอร์เอง ไม่เข้าไปดูข้างใน |
| `-i` | **I**node | แสดงเลข inode ของไฟล์ |

**คอมโบยอดฮิต (flag เขียนติดกันได้):**
- `ls -l` → รายละเอียดเต็ม
- `ls -la` (= `ls -al`) → เห็นทุกอย่างรวมไฟล์ซ่อน แบบยาว
- `ls -lh` → ขนาดอ่านง่าย (KB/MB แทน byte)
- `ls -lah` → รวมหมด: ยาว + ซ่อน + ขนาดอ่านง่าย
- `ls -ltr` → เรียงเวลา เก่า→ใหม่ (ไฟล์ล่าสุดอยู่ล่างสุด) ← ใช้บ่อยมากตอนดู log
- `ls -lS` → ไฟล์ใหญ่อยู่บนสุด (หาไฟล์กินพื้นที่)

> 💡 `ls -l -a -h` กับ `ls -lah` คือคำสั่งเดียวกัน — flag ตัวอักษรเดี่ยวเขียนรวมกันได้

**File Management (บท 03)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `cat` | con**CAT**enate | ต่อ/แสดงเนื้อหาไฟล์ |
| `cp` | **C**o**P**y | คัดลอกไฟล์ |
| `mv` | **M**o**V**e | ย้าย หรือเปลี่ยนชื่อไฟล์ |
| `mkdir` | **M**a**K**e **DIR**ectory | สร้างโฟลเดอร์ใหม่ |
| `rm` | **R**e**M**ove | ลบ (⚠️ ไม่มีถังขยะ) |

**Viewing Files (บท 04)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `head` | head (หัว) | ดูบรรทัดแรก ๆ ของไฟล์ |
| `tail` | tail (หาง) | ดูบรรทัดท้าย ๆ ของไฟล์ |
| `wc` | **W**ord **C**ount | นับบรรทัด/คำ/ตัวอักษร |
| `less` | เล่นคำกับ `more` ("less is more") | เปิดไฟล์ดูทีละหน้า เลื่อนได้ (ออกด้วย `q`) |

**Searching (บท 05)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `grep` | **G**lobal **R**egular **E**xpression **P**rint | ค้นหา "ข้อความในไฟล์" |
| `find` | find (ค้นหา) | ค้นหา "ตัวไฟล์/โฟลเดอร์" |

**Pipe & Redirect (บท 06)**

| สัญลักษณ์/คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `\|` | pipe (ท่อ) | ส่ง output ของคำสั่งซ้าย → เป็น input ของคำสั่งขวา |
| `>` | redirect | เขียน output ลงไฟล์ (**ทับ** ของเดิม) |
| `>>` | append | เขียน output **ต่อท้าย** ไฟล์ |
| `cut` | cut (ตัด) | ตัดเอาเฉพาะคอลัมน์ที่ต้องการ |
| `sort` | sort (เรียง) | เรียงลำดับบรรทัด |
| `uniq` | **UNIQ**ue | ยุบบรรทัดซ้ำที่ติดกัน (นับด้วย `-c`) |

**Environment & Scripts (บท 07)**

| คำสั่ง | ย่อมาจาก | ความหมาย |
|--------|----------|----------|
| `source` (`.`) | source (แหล่ง) | รัน script ใน shell **ปัจจุบัน** → ตัวแปรอยู่ต่อ |
| `chmod` | **CH**ange **MOD**e | เปลี่ยนสิทธิ์ไฟล์ (เช่น `+x` ให้รันได้) |
| `export` | export | ส่งตัวแปรให้ process ลูกมองเห็น |

**Flags / ตัวเลือกที่เจอบ่อย (เติมท้ายคำสั่ง)**

| Flag | ย่อมาจาก | ความหมาย |
|------|----------|----------|
| `-r` / `-R` | **R**ecursive | ทำซ้ำลงไปทุกชั้นของโฟลเดอร์ (`cp -r`, `grep -r`) |
| `-n` | **N**umber | `grep -n` โชว์เลขบรรทัด · `tail -n N` จำนวนบรรทัด |
| `-c` | **C**ount | นับจำนวน (`grep -c`, `uniq -c`) |
| `-l` | **L**ist files | `grep -l` โชว์เฉพาะ "ชื่อไฟล์" ที่เจอ |
| `-i` | **I**nteractive / **I**gnore-case | `rm -i` ถามก่อนลบ · `grep -i` ไม่สนตัวพิมพ์ |
| `-f` | **F**ollow / **F**ield | `tail -f` ตามดู log สด · `cut -f` เลือกคอลัมน์ |
| `-d` | **D**elimiter | `cut -d','` ใช้ `,` เป็นตัวคั่น |
| `-L` | **L**evel | `tree -L 2` แสดงลึก 2 ชั้น |
| `-v` | in**V**ert | `grep -v` กลับเงื่อนไข (เอาบรรทัดที่ **ไม่** match) |
| `+x` | e**X**ecute | `chmod +x` ทำให้ไฟล์รันได้ |

> 📌 `tail -n +2` = เริ่มอ่านจากบรรทัดที่ 2 (เครื่องหมาย `+` แปลว่า "ตั้งแต่บรรทัดนี้เป็นต้นไป")
> ใช้บ่อยตอนตัด header ออกก่อนนับข้อมูล (เห็นในบท 06)

>

>

>

>

**เฉลย Activity:**
```bash
pwd
cd projects/ecommerce && pwd
cd ../../   # หรือ cd /.../02_navigation
tree -L 2
```

```
.
├── archive
└── projects
    ├── banking
    └── ecommerce
```

- ไม่มี `tree` ติดตั้ง → ให้ใช้ `ls -R` แทนชั่วคราว
- พิมพ์ absolute path ผิดเพราะ copy path มาจากเครื่องอื่น
**Teaching tip:** ให้นักเรียนวาดแผนผัง folder บนกระดาษก่อน `cd` จริง

### บทที่ 03 — File Management

#### 🎤 Speaker Notes (ผู้สอน)
> ทีนี้เรารู้แล้วว่าเราอยู่ไหน เดินไปไหนได้
>
> งานจริงของ Data Engineer เริ่มตรงนี้ครับ — เราต้อง "จัดการไฟล์ข้อมูล"
>
> สมมติพี่ในทีมบอกว่า "ไฟล์ raw มาแล้วนะ ช่วย backup ไว้ก่อน"
>
> *(เว้นจังหวะ)*
>
> ผมอยากถามว่า ทำไมต้อง backup ก่อนแตะของจริง?
>
> *(รอฟังคำตอบ)*
>
> เพราะบน Linux… ตรงนี้สำคัญมาก ฟังดี ๆ นะครับ
>
> **`rm` ไม่มีถังขยะ**
>
> ลบแล้วคือหายเลย ไม่มี Recycle Bin ให้กู้
>
> เพราะงั้นกฎเหล็กข้อแรกของห้องนี้คือ — มองให้ดีก่อนกด Enter ทุกครั้ง
>
> วันนี้เราจะใช้ `cp` คัดลอก, `mv` ย้าย/เปลี่ยนชื่อ, `mkdir` สร้างโฟลเดอร์
>
> ส่วน `rm` เราจะใช้แบบระมัดระวัง และผมจะสอน `rm -i` ที่ถามยืนยันก่อนลบ

**เฉลย Activity:**
```bash
cp raw/*.csv backup/
ls backup/        # ต้องเห็น 3 ไฟล์
```
**Expected:** `customer.csv  product.csv  sales.csv`
**นักเรียนมักพลาด:**
- ลืม `-r` ตอน `cp` ทั้ง folder
- ใช้ `rm` ผิด path ⚠️ — **ย้ำว่า Linux ไม่มีถังขยะ**
- `cp raw/ backup/` (ไม่มี `*` และไม่มี `-r`) → error
**Teaching tip (สำคัญด้านความปลอดภัย):** สอน `rm -i` (ถามยืนยันทีละไฟล์) และเตือนเรื่อง `rm -rf /`
ห้ามให้นักเรียนทดลอง `rm -rf` กับ path ที่ขึ้นต้นด้วย `/` หรือ `~` เด็ดขาด

### บทที่ 04 — Viewing Files

#### 🎤 Speaker Notes (ผู้สอน)

>
> "ถ้าไฟล์มีล้านบรรทัด การ cat ทั้งไฟล์ดีไหม?"
>
> *(รอฟังคำตอบ)*
>
> ลองนึกภาพครับ ไฟล์ log จริงในงาน บางทีเป็นล้านบรรทัด
>
> ถ้าเรา `cat` ทั้งไฟล์ หน้าจอจะวิ่งจนตาลาย แล้วก็ไม่ได้อะไรเลย
>
> *(เว้นจังหวะ)*
>
> มืออาชีพไม่อ่านทั้งไฟล์ครับ — เขาเลือกอ่านเฉพาะส่วนที่ต้องการ
>
> อยากดูหัวไฟล์ว่า column อะไรบ้าง → `head`
>
> อยากดูท้ายไฟล์ว่าเกิดอะไรล่าสุด → `tail`
>
> และที่เด็ดสุด เวลา pipeline กำลังรันอยู่ เราอยากดู log วิ่งสด ๆ → `tail -f`
>
> เดี๋ยวผมจะสาธิตให้ดู ผมจะเปิดสองจอ จอนึงเขียน log เข้าไป อีกจอเฝ้าดู
>
> *(สาธิต tail -f สด)*
>
> อ้อ เตือนไว้ก่อนนะครับ ถ้าใครติดอยู่ใน `less` ออกไม่ได้ — กด `q`
>
> ถ้า `tail -f` ค้าง — กด `Ctrl + C`

**เฉลย Activity:**
```bash
wc -l big_file.csv          # 501
head -n 1 big_file.csv      # ดู header
```
**Expected (ยืนยันแล้ว):**
```
501 big_file.csv
event_id,user_id,event_type,timestamp,value
```
**โน้ต:** `big_file.csv` ถูก generate แบบสุ่ม ค่าในแต่ละแถวจะต่างกันในแต่ละเครื่อง
แต่ **จำนวนบรรทัด (501) และ header เหมือนกันเสมอ**
**นักเรียนมักพลาด:**
- ติดอยู่ใน `less` ออกไม่ได้ → บอกล่วงหน้าว่า **กด `q`**
- `tail -f` ค้าง → บอกว่า **กด `Ctrl+C`** เพื่อหยุด
**Teaching tip:** สาธิต `tail -f` สดโดยเปิด 2 terminal — terminal หนึ่ง `echo "...">> application.log`
อีก terminal `tail -f application.log` ให้เห็นบรรทัดวิ่งเข้ามา

### บทที่ 05 — Searching

#### 🎤 Speaker Notes (ผู้สอน)
> เอาล่ะครับ มาถึงทักษะที่ทุกคนจะใช้แทบทุกวันในงานจริง
>
> สมมตินะครับ ตี 2 มี alert เด้งเข้า Teams บอกว่า pipeline ล่ม
>
> พี่ในทีมโยน log มาให้กองนึง บอกว่า "หาให้หน่อยว่ามี error กี่ตัว เกิดตรงไหน"
>
> *(เว้นจังหวะ)*
>
> ทีนี้ ถ้าเราไล่อ่านทีละบรรทัดด้วยตา…
>
> ผมถามจริง ๆ ครับ — มันจะทันไหม แล้วจะพลาดไหม?
>
> *(รอฟังคำตอบ)*
>
> ช้าแน่นอน แล้วก็พลาดง่ายมาก
>
> เครื่องมือของเราชื่อ `grep` ครับ — มันคือการ "ค้นหาข้อความในไฟล์"
>
> `grep ERROR` ก็คือบอกว่า "ขอเฉพาะบรรทัดที่มีคำว่า ERROR"
>
> เติม `-r` ค้นทั้งโฟลเดอร์, เติม `-n` บอกเลขบรรทัด, เติม `-c` นับให้เลย
>
> *(ย้ำตรงนี้)* เดี๋ยวตอนเฉลย คำตอบคือ **10** นะครับ ไม่ใช่ 9
>
> ใครนับมือแล้วได้ 9 แปลว่ามองข้ามไป 1 บรรทัด — นี่แหละครับเหตุผลที่เราใช้คำสั่ง ไม่ใช้ตา
>
> และอย่าสับสนนะครับ `grep` หา "ข้อความในไฟล์" ส่วน `find` หา "ตัวไฟล์เอง"

**เฉลย Activity:**
```bash
grep -r "ERROR" logs/ | wc -l     # 10
grep -rl "timeout" logs/          # logs/pipeline.log
```
**Expected (ยืนยันแล้ว):**
```
10
logs/pipeline.log
```
> ⚠️ **จุดที่ต้องระวังในการเฉลย:** คำตอบคือ **10** ไม่ใช่ 9
> (app.log 2 + error.log 6 + pipeline.log 2 = 10) — นักเรียนบางคนนับมือแล้วได้ 9 เพราะมองข้าม
> ให้ใช้คำสั่งยืนยันเสมอ
**นักเรียนมักพลาด:**
- สับสน `grep` (หาในไฟล์) กับ `find` (หาตัวไฟล์)
- ลืม `-r` ตอนค้นทั้ง folder → grep ฟ้อง "Is a directory"
- ใช้ `-l` (แสดงชื่อไฟล์) สลับกับ `-c` (นับ)
**คำถามนำสนทนา:** "ถ้าอยากนับว่าแต่ละไฟล์มี ERROR กี่ครั้งแยกกัน ใช้อะไร?" → `grep -rc ERROR logs/`

### บทที่ 06 — Pipe & Redirect (บทที่ยากที่สุด)

#### 🎤 Speaker Notes (ผู้สอน)
> บทนี้เป็นบทที่ยากที่สุดของวันนี้ครับ แต่ก็เป็น "เวทมนตร์" ของ Linux เลย
>
> เมื่อกี้เราเรียน grep หา error ได้แล้ว
>
> ทีนี้ ถ้าผมไม่อยากแค่ "เห็น" error แต่อยาก "นับ" ว่ามีกี่ตัวล่ะ?
>
> *(เว้นจังหวะ)*
>
> นี่คือจุดที่ pipe เข้ามา — เครื่องหมาย `|` ขีดตั้งนี้
>
> ผมอยากให้คิดภาพ "ท่อน้ำ" ครับ
>
> ผลลัพธ์ที่ไหลออกจากคำสั่งซ้าย ไหลผ่านท่อ เข้าไปเป็นวัตถุดิบของคำสั่งขวา
>
> `grep ERROR | wc -l` ก็คือ "หา error ออกมา แล้วส่งไปนับจำนวนบรรทัด"
>
> *(เว้นจังหวะ)*
>
> นี่คือปรัชญาของ Unix เลยครับ — โปรแกรมเล็ก ๆ แต่ละตัวทำหน้าที่เดียวให้เก่ง
>
> แล้วเราเอามันมาต่อท่อกันเป็นงานใหญ่
>
> อีกอย่างที่ต้องระวังมาก ๆ คือการเขียนผลลัพธ์ลงไฟล์
>
> `>` หนึ่งตัว = เขียนทับของเดิมหายหมด
>
> `>>` สองตัว = ต่อท้าย ของเดิมอยู่ครบ
>
> *(ย้ำ)* พิมพ์ `>` ผิดเป็น `>>` ข้อมูลเก่าหายทันทีโดยไม่เตือนนะครับ ระวังตรงนี้ให้ดี
>
> เดี๋ยวเราจะค่อย ๆ ต่อ pipeline ทีละขั้น ให้เห็นว่า output เปลี่ยนยังไงในแต่ละสเต็ป

**เฉลย Activity:**
```bash
cut -d',' -f4 input/users.csv | tail -n +2 | sort | uniq -c > output/report.txt
cat output/report.txt
```
**Expected (ยืนยันแล้ว — ลำดับขึ้นกับ sort):**
```
   3 enterprise
   7 free
   5 pro
```
**โน้ตเฉลย:**
- `tail -n +2` ตัด header ก่อนนับ — ถ้าลืม จะมีบรรทัด `1 plan` โผล่มา
- TH count = 8 (`grep -c TH input/users.csv`)
**นักเรียนมักพลาด (เยอะที่สุดในบทนี้):**
- ใช้ `>` แทน `>>` แล้วเขียนทับข้อมูลเดิมหาย → **ย้ำความต่าง > กับ >>**
- ลืม `-d','` ทำให้ `cut` ตัดผิด (default delimiter คือ TAB)
- เรียงลำดับ pipe ผิด (เช่น `uniq` ก่อน `sort` → นับผิดเพราะ uniq นับเฉพาะที่ติดกัน)
**Teaching tip:** อธิบาย Unix philosophy — "โปรแกรมเล็กทำหน้าที่เดียวให้ดี แล้วต่อท่อกัน"
ให้นักเรียนสร้าง pipeline ทีละขั้น เพิ่มทีละคำสั่ง เห็น output เปลี่ยนทุกขั้น

**คำถามนำสนทนา:** "ทำไม password ไม่ควรอยู่ในโค้ด?" → โยงไป `.env`, `.gitignore`, security
